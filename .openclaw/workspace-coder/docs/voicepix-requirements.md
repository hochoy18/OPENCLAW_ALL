## 1. 功能概览

### 1.1 产品定位

VoicePix 是一款基于 MiniMax Speech API 的网页端语音合成工具，聚焦于**文字转语音（T2A）** 和**声音克隆（Voice Clone）** 两个核心场景，为内容创作者、企业用户和开发者提供快速、高质量的语音生成能力。

### 1.2 核心价值

- **零门槛配音**：无需专业设备，输入文字即可生成专业语音
- **声音个性化**：支持声音克隆，打造专属音色
- **纯前端部署**：单 HTML 文件，无后端依赖，打开即用
- **隐私优先**：API Key 仅存储在浏览器 localStorage，不上传任何用户数据

### 1.3 现有 MVP 功能总结

| 模块 | 功能点 | 现状 |
|------|--------|------|
| **T2A** | 文字输入（≤10,000 字） | ✅ 已实现 |
| **T2A** | 19 种预设音色（10 中文 + 9 英文） | ✅ 已实现 |
| **T2A** | 音色预览播放 | ✅ 已实现 |
| **T2A** | 语速调节（0.5x ~ 2.0x） | ✅ 已实现 |
| **T2A** | 音调调节（-12 ~ +12） | ✅ 已实现 |
| **T2A** | 输出格式（MP3 / PCM / WAV / FLAC） | ✅ 已实现 |
| **T2A** | 生成音频在线播放 + 下载 | ✅ 已实现 |
| **Clone** | 上传音频克隆（MP3/M4A/WAV，10s~5min，≤20MB） | ✅ 已实现 |
| **Clone** | 自定义 voice_id | ✅ 已实现 |
| **Clone** | 克隆成功后自动加入音色选择器 | ✅ 已实现 |
| **通用** | 中英双语界面切换 | ✅ 已实现 |
| **通用** | API Key 本地存储 | ✅ 已实现 |
| **通用** | 深色主题 UI | ✅ 已实现 |

## 2. 5W1H 需求分析

### 2.1 Who（目标用户）

| 用户群体 | 描述 | 典型场景 |
|----------|------|----------|
| **内容创作者** | 视频 UP 主、播客主、有声书制作者 | 为视频/播客快速生成配音 |
| **企业用户** | 营销团队、培训部门、产品团队 | 产品介绍视频配音、培训材料语音化 |
| **教育从业者** | 语言学习、在线教育平台 | 外语跟读练习、教材音频化 |
| **开发者** | 需要集成 TTS 能力的应用开发者 | 将 VoicePix 作为 Demo 或直接集成 |
| **个人用户** | 日常趣味语音生成 | 个性化语音祝福、练声 |

### 2.2 What（需求本质）

将**文字内容**和**参考音频**转化为**高质量语音输出**，核心是解决"快速获得专业配音"的问题。

### 2.3 Why（解决的问题）

| 痛点 | VoicePix 如何解决 |
|------|-------------------|
| 专业配音成本高、周期长 | 输入文字，秒级生成配音 |
| 需要特定音色但无录音条件 | 声音克隆，用少量样本复制音色 |
| TTS 工具操作复杂、需要安装 | 纯网页，打开即用，无需安装 |
| 跨语言配音困难 | 内置中英双语界面 + 19 种预设音色 |

### 2.4 Where（使用场景）

- **视频制作**：B 站视频、抖音/快手短视频、企业宣传片
- **音频内容**：播客、有声书、语音博客
- **教育培训**：课件配音、在线课程、跟读练习
- **产品演示**：App 操作指引、产品功能介绍
- **无障碍**：为视障用户提供文字转语音服务

### 2.5 When（使用时机）

- 需要快速验证配音效果时（原型阶段）
- 内容已完成文字撰写，需要快速配音时
- 无专业录音环境和设备时
- 需要大量不同音色进行 A/B 测试时

### 2.6 How（实现方式）

- **前端**：纯 HTML + CSS + JavaScript，单文件部署
- **API 调用**：MiniMax T2A v2 API（`speech-02` / `speech-02-hd` 模型）、Voice Clone API
- **存储**：localStorage 存储 API Key，无服务端依赖
- **主题**：深色主题优先，支持中英双语

## 3. 用户故事（User Stories）

### 3.1 T2A 核心流程

```
作为一名【视频 UP 主】
我想要【输入文字并选择音色，生成配音】
以便【快速为我的视频生成开场白，无需录音设备和专业人员】

Given 我已设置有效的 API Key
When  我输入一段文字，选择一个中文音色，并点击"生成语音"
Then  系统返回对应音频，可在线播放并下载文件
```

```
作为一名【英语学习者】
我想要【用不同英文音色播放我的学习内容】
以便【练习听力和跟读，找到最喜欢的发音风格】

Given 我已设置有效的 API Key
When  我输入英语句子，选择一个英文音色并预览
Then  系统播放对应音色的语音，我可以反复听读
```

### 3.2 声音克隆流程

```
作为一名【自媒体创作者】
我想要【上传自己的声音样本，克隆专属音色】
以便【用我的声音为视频配音，保持个人风格】

Given 我已设置有效的 API Key 且有一份 30 秒的人声 MP3
When  我上传音频，设置自定义 voice_id 并点击"开始克隆"
Then  克隆成功后，该 voice_id 自动出现在音色选择器中
```

### 3.3 配置管理流程

```
作为一名【隐私敏感的用户】
我想要【API Key 仅保存在本地浏览器】
以便【不担心密钥泄露给第三方】

Given 我首次使用 VoicePix
When  我在 API Key 设置弹窗中输入 Group ID 和 Secret
Then  系统将密钥存入 localStorage，不经过任何服务器
```

## 4. EPIC 分解与扩展方向

> 将 VoicePix 现有 MVP 视为 **EPIC-0：核心语音生成**，以下为 EPIC-1 到 EPIC-7 的扩展方向。

```
VoicePix 产品规划
│
├── EPIC-0：核心语音生成 [当前MVP] ✅
│   ├── REQ-0.1：文字转语音（T2A）
│   ├── REQ-0.2：声音克隆（Voice Clone）
│   └── REQ-0.3：API Key 本地配置
│
├── EPIC-1：内容创作效率提升 ⭐
│   ├── REQ-1.1：长文本自动分片（突破 10,000 字限制）
│   ├── REQ-1.2：SSML 标记支持（精细控制停顿、发音）
│   ├── REQ-1.3：分角色配音（为不同角色分配不同音色）
│   └── REQ-1.4：音频时长预估（生成前显示预估时长）
│
├── EPIC-2：声音资产管理 🗣
│   ├── REQ-2.1：克隆声音库管理（增删改查已克隆声音）
│   ├── REQ-2.2：克隆声音变体生成（同一音色多种风格）
│   ├── REQ-2.3：声音收藏/置顶常用音色
│   └── REQ-2.4：声音分享（导出/导入 voice_id 配置）
│
├── EPIC-3：工作流与协作 👥
│   ├── REQ-3.1：生成历史记录（本地记录历史音频）
│   ├── REQ-3.2：项目文件夹（分组管理不同用途的音频）
│   ├── REQ-3.3：音频命名与标签（方便检索）
│   └── REQ-3.4：一键分享（生成分享链接）
│
├── EPIC-4：内容导入与集成 📄
│   ├── REQ-4.1：Word 文档导入（.docx）
│   ├── REQ-4.2：PDF 文本提取 + T2A
│   ├── REQ-4.3：视频字幕轨道提取 + 配音
│   └── REQ-4.4：批量文本导入（CSV/多段落）
│
├── EPIC-5：平台与分发 🌐
│   ├── REQ-5.1：响应式移动端适配
│   ├── REQ-5.2：PWA 离线支持
│   ├── REQ-5.3：Chrome 扩展（选中文本右键生成语音）
│   └── REQ-5.4：开放 API（供第三方应用调用）
│
├── EPIC-6：高级语音控制 🎛
│   ├── REQ-6.1：多音字/读音纠正（自定义读音规则）
│   ├── REQ-6.2：情感控制（高兴/悲伤/平静等模式）
│   ├── REQ-6.3：背景音乐混音（配音 + BGM 叠加）
│   └── REQ-6.4：混响/均衡器音效调节
│
└── EPIC-7：安全合规与多租户 🔐
    ├── REQ-7.1：API Key 加密存储（Web Crypto API）
    ├── REQ-7.2：使用量统计（本地记录 API 调用量）
    ├── REQ-7.3：团队 Key 管理（多人共享 Key 配置）
    └── REQ-7.4：数据导出/删除（GDPR 合规）
```

### EPIC 详细说明

#### EPIC-1：内容创作效率提升
解决长文本用户和需要精细控制的专业用户的需求。

#### EPIC-2：声音资产管理
帮助用户管理日益增多的克隆音色和常用音色，适合重度创作者。

#### EPIC-3：工作流与协作
引入项目管理和历史功能，适合企业场景和多项目并行。

#### EPIC-4：内容导入与集成
打通内容生产上游，用户可以直接处理已有的 Word/PDF/视频素材。

#### EPIC-5：平台与分发
移动端适配和 PWA 支持覆盖更多使用场景，Chrome 扩展提升操作效率。

#### EPIC-6：高级语音控制
面向专业用户，提供情感控制和音效调节等高级功能。

#### EPIC-7：安全合规与多租户
满足企业级安全和合规需求。

## 6. 验收标准（Acceptance Criteria）

### 6.1 T2A 核心功能

```
Given 用户已设置有效的 API Key 且选择了音色
When  用户输入不超过 10,000 字的中文或英文文本，点击"生成语音"
Then  系统在 30 秒内返回对应音频文件（MP3/WAV/PCM/FLAC）
      音频可在线播放，进度条可拖动
      下载按钮可正确下载对应格式文件
```

```
Given 用户选择了音色并点击预览按钮
When  系统播放该音色的简短示例语音（"你好，这是一条语音预览"）
Then  播放流畅，音色与标签描述一致
      预览过程中按钮显示加载状态，结束后恢复
```

### 6.2 声音克隆功能

```
Given 用户上传了 30 秒、15MB 的 MP3 音频文件，并设置了自定义 voice_id
When  点击"开始克隆"，等待克隆完成
Then  克隆成功后，新 voice_id 自动出现在 T2A 音色选择器中
      用户可在 T2A 中使用该音色生成语音
```

```
Given 用户上传的文件超过 20MB 或时长不足 10 秒
When  用户点击上传或拖放文件
Then  系统显示友好的错误提示，说明限制条件
      文件不被接受，不触发 API 调用
```

### 6.3 界面与多语言

```
Given 用户在中文界面下操作
When  用户点击语言切换为 English
Then  页面所有标签、按钮文字、placeholder 替换为英文
      音色预览文本变为英文"Hello, this is a voice preview."
```

### 6.4 API Key 管理

```
Given 用户首次打开页面，未设置 API Key
When  用户点击任何需要 API 的操作（T2A生成/克隆/预览）
Then  系统弹出 API Key 设置弹窗，提示用户输入
```

```
Given 用户输入了 API Key 并保存
When  关闭弹窗后刷新页面
Then  API Key 自动从 localStorage 恢复，无需重新输入
```

### 6.5 长文本分片（EPIC-1 - Must 新增）

```
Given 用户输入了 15,000 字文本
When  点击"生成语音"
Then  系统自动将文本分片为多个 ≤10,000 字的片段
      按顺序调用 T2A API，合并所有片段音频
      合并完成后提供完整音频的播放和下载
```

### 6.6 移动端适配（EPIC-5 - Should 新增）

```
Given 用户在 375px 宽度的 iPhone 上访问 VoicePix
When  页面加载完成
Then  所有元素（按钮、输入框、音色网格）正常显示
      无横向溢出，无需左右滑动
      Tab 切换、弹窗、播放器均正常工作
```

## 7. 非功能性需求

| 维度 | 要求 |
|------|------|
| **性能** | T2A API 调用到返回音频 < 30 秒（10,000 字以内） |
| **可用性** | 页面加载时间 < 3 秒（2G 网络参考值） |
| **兼容性** | 主流浏览器 Chrome/Firefox/Safari/Edge 最新两个版本 |
| **安全性** | API Key 仅存储在 localStorage，不发送至第三方 |
| **隐私** | 不收集任何用户音频或文本数据 |
| **可访问性** | 支持键盘导航，主要交互元素有适当对比度 |

## 9. 风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| MiniMax API 调佣失败或响应慢 | 用户体验下降 | 添加超时提示和重试机制 |
| localStorage 存储达到上限 | Key 丢失 | 提供 Key 导出/导入功能 |
| 声音克隆质量依赖样本质量 | 克隆音色不理想 | 在 UI 中提供样本质量指导 |
| 10,000 字限制影响长文本用户 | 用户流失 | 优先实现长文本分片功能 |
| 移动端体验差 | 用户流失 | 优先实现移动端适配 |
