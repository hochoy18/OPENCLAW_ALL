# 全流程验收标准

说明：1. 所有【】内为可替换内容，替换后删除【】；2. 复制全部HTML代码，直接粘贴到公众号后台“源码”模式，再切换回“编辑”模式微调即可；3. 严格遵循此前规范，无需修改样式参数；4. 下方验收标准对应发稿全流程，审核时逐步核对，全部通过方可进入下一步。

## 一、AI科技类公众号发稿全流程验收标准（审核必查）

### 阶段1：热点采集验收标准（必过）

**必须执行动作：**

- 通读所有热点信息源（如行业官网、权威媒体、官方公告、技术社区），确认AI/科技类热点的真实性、时效性；

- 核对热点相关的技术参数、模型版本、发布主体、核心数据等关键信息，确保无偏差；

- 筛查热点内容是否存在敏感词、合规风险（如涉政、夸大、侵权）及竞品纠纷；

- 输出结构化热点素材包（含信息源链接、核心亮点、适配账号定位分析）。

**验收通过标准：**

- 热点信息可溯源，无事实错误、无过时内容；

- 无违规、无擦边、无夸大表述，符合账号AI/科技类定位；

- 素材包完整，关键技术信息准确无误。

### 阶段2：选题验收标准（必过）

**必须执行动作：**

- 结合热点素材、账号定位，确定选题方向，明确选题核心（如技术解析、实操教程、对比测评）；

- 撰写选题大纲，明确文章结构（导语→核心章节→总结→引导），标注各章节核心内容；

- 确认选题无重复（查询账号历史发文），且具有可读性、实用性（贴合目标读者需求）。

**验收通过标准：**

- 选题贴合AI/科技热点，适配账号定位，无重复、无冗余；

- 大纲结构清晰，章节逻辑连贯，核心内容明确；

- 选题具有实用性，能为读者提供技术价值、信息价值。

### 阶段3：写文（Markdown初稿）验收标准（必过）

**必须执行动作：**

- 按照选题大纲撰写初稿，严格遵循字数规范，术语统一，不前后矛盾；

- 核心技术点、关键结论用加粗标注，代码、命令、参数准确可复现；

- 段落简洁，手机端单段≤3行、≤120字，连续纯文字不超过400字（需预留配图位置）；

- 导语引出核心内容，结尾提炼总结，逻辑闭环。

**验收通过标准：**

- 字数达标（正文≥800字），结构完整，符合大纲要求；

- 无事实错误、无逻辑漏洞，技术术语统一、语句通顺；

- 代码/命令可复现，无乱写、无错误；

- 无错别字、无病句，标点符号使用规范。

### 阶段4：审核内容验收标准（必过）

**必须执行动作：**

- 全文通读，检查错别字、病句、标点错误，修正语句不通顺之处；

- 核对技术数据、模型版本、价格、API规则等关键信息，确保准确无误；

- 筛查敏感词、合规风险（如涉政、低俗、侵权），删除违规内容；

- 检查配图规划（每300-400字1张图），确认图片类型贴合内容（架构图/流程图/截图优先）；

- 检查内链、参考资料链接，确保可正常打开，无失效链接。

**验收通过标准：**

- 0错别字、0事实错误、0合规风险、0版权风险；

- 技术参数、版本号、价格等信息准确无误；

- 配图规划合理，符合配图规范，无水印、无侵权风险；

- 链接可正常访问，参考资料真实可溯源。

### 阶段5：发布草稿箱验收标准（必过）

**必须执行动作：**

- 将排版好的HTML内容粘贴到公众号后台，保存为草稿箱；

- 进行手机端、PC端双重预览，检查格式、排版、图片、代码块是否正常；

- 设置文章相关信息（标题、摘要、作者、封面图、原创声明、留言权限）；

- 再次通读草稿，确认无遗漏、无错误，排版符合规范。

**验收通过标准：**

- 草稿箱内容完整，无乱码、无排版错位，代码块可正常显示、横向可滚动；

- 标题（14-26字）、摘要（60-120字）符合规范，手动填写无敷衍；

- 封面图（900×383px，≤200KB）、正文图（1080px宽，≤2MB）符合配图规范；

- 原创声明、作者、留言权限设置正确，手机/PC预览无任何问题；

- 可直接提交群发，无任何待修正问题。
