## ⚠️ 微信公众号文章铁律（2026-03-27 教训）

### 绝对禁止的行为（一次都不行）

1. **不允许复杂CSS**：只能用极简HTML标签
   - 可用：`<h1>` `<h2>` `<h3>` `<p>` `<ul>` `<li>` `<strong>` `<img>` `<section>`
   - 禁止：任何 style 属性里有 `margin`, `padding`, `font-size`, `color`, `line-height` 等

2. **不允许图片堆在一起**：每篇必须3张，均匀分布
   - 第1张：文章开头（第一个 h2 之后）
   - 第2张：文章中间（第三个 h2 之后）
   - 第3张：文章结尾（第五个 h2 之后）

3. **不允许推送前不检查**：必须自查3遍再交付
   - 检查1：标题和结构是否正确
   - 检查2：图片是否在正确位置
   - 检查3：HTML是否干净无乱码

4. **不允许配图和文章内容无关**：AI测评用AI图，职场内容用职场图

### 正确的文章生产流程

```
写 Markdown → 转极简HTML → 插入3张配图（开头/中间/结尾）
→ 自查3遍 → 推送草稿 → 通知用户
```

### 违规后果

- 用户投诉格式问题 = 工作失误
- 重复推送超过2次 = 严重失误
- 如果再有第三次 = 失职
