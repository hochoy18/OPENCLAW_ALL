一个GitHub项目，让Claude Code学会了"团队作战"

作者：AI Coding 观察员 | 来源：GitHub 热榜

一句话概括
Claude Code 是一个 AI 编程助手，但通常只能单打独斗。oh-my-claudecode 让它变成了一个工程团队——有架构师定方案、有执行者写代码、有测试者找 Bug，各司其职，协同作战。

过去一年，Claude Code、Copilot 这些 AI 编程工具已经很普遍了。但大多数人的用法是：一个窗口，AI 帮你补全代码。

这就好比你请了一个很厉害的程序员，但他只能一个人干活，拆不了任务、协作不了分工。

多Agent协作：多个AI Agent同时工作，各司其职

最近在 GitHub 上发现了一个项目，叫 oh-my-claudecode。 Stars 数已经超过 19k，上了 GitHub Trending。它解决的就是这个问题——让 Claude Code 从"单打独斗"变成"团队作战"。

一、一个指令，自动拆解、自动执行

传统用法：你给 Claude Code 一个任务，它就闷头帮你写代码。

oh-my-claudecode 的用法：你说一句话，它自动拆解成多个子任务，分配给不同的 Agent 并行执行。

比如输入：

/autopilot: build a REST API for managing tasks

它会自动规划出：数据模型怎么设计、API 路由怎么写、谁来写测试用例……然后分给不同的 Agent 同时开工，最后汇总结果。

1. 团队流水线：五步自动闭环

oh-my-claudecode 把协作流程拆成五个阶段：

五步闭环：PLAN → PRD → EXEC → VERIFY → FIX

  • PLAN：分析需求，制定执行计划
  • PRD：输出产品需求文档
  • EXEC：分工编码，各模块同步开发
  • VERIFY：测试验证，检查代码质量
  • FIX：自动修复发现的问题

这和真实工程团队的分工几乎一样——有 PM 写需求、有开发者写代码、有 QA 测功能。

2. 多实例并行：Claude Code + Codex + Gemini

除了 Claude Code，oh-my-claudecode 还能同时调度 Codex（OpenAI）和 Gemini（Google）的 Agent 实例。

不同模型擅长不同领域：Codex 适合代码审查和安全分析，Gemini 适合 UI 设计和文档编写，Claude Code 擅长通用编程任务。

二、安装只需一条命令

oh-my-claudecode 的安装很简单，不需要学新语法：

/plugin marketplace add https://github.com/Yeachan-Heo/oh-my-claudecode

安装完成后，常用指令包括：

  • /autopilot：描述你的需求，自动拆解执行
  • /deep-interview：用苏格拉底式提问帮你澄清需求，再动手
  • /team N:executor "任务"：启动 N 个并行 Agent 执行特定任务

一个开发者，多个 AI Agent 同时协作

三、适合哪些场景？

oh-my-claudecode 不是银弹。它最适合这几类场景：

  • 大型重构项目：多模块并行改写，比一个人逐个改效率高得多
  • 安全代码审计：多个 Agent 分工检查不同模块，比单一审计更全面
  • PR Review：Codex 审查架构，Gemini 检查 UI，Claude Code 验证逻辑
  • 复杂需求澄清：用 /deep-interview 功能在动手前先理清思路

总结

oh-my-claudecode 解决的是一个很实在的问题：AI 编程工具很强，但一个人干活有上限。当你能指挥一个团队，各有所长、并行开工，效率就不再是线性增长，而是协同放大。

这个项目目前 Stars 突破 19k，说明社区对"多 Agent 协同"这个方向的认可。如果你也在用 Claude Code 或 Codex，不妨试试这个扩展，说不定会打开新世界。
