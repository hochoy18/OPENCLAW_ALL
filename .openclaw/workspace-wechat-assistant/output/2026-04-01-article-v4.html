一个开源项目，让Claude Code学会了团队作战

作者：AI Coding 观察员 | 来源：GitHub 热榜

Claude Code 是一个很能写的 AI 编程助手，但大多数人的用法是"一个窗口，AI 帮你补全代码"——这就好比你请了一个厉害的程序员，但他只能一个人干活，拆不了任务、协作不了分工。

最近在 GitHub 上发现了一个开源项目 oh-my-claudecode，Stars 突破 19k。它解决的就是这个问题：让 Claude Code 从单打独斗，变成一个工程团队——有架构师定方案、有执行者写代码、有测试者找 Bug，各司其职，协同作战。

图 | oh-my-claudecode：让Claude Code团队协作的核心项目

## 一、一个指令，自动拆解、自动执行 ### 1. 团队流水线：五步自动闭环

oh-my-claudecode 把协作流程拆成五个阶段，形成自动闭环：

  • PLAN：分析需求，制定执行计划
  • PRD：输出产品需求文档
  • EXEC：分工编码，各模块同步开发
  • VERIFY：测试验证，检查代码质量
  • FIX：自动修复发现的问题

图 | 五步闭环：PLAN → PRD → EXEC → VERIFY → FIX

这和真实工程团队的分工几乎一样——有 PM 写需求、有开发者写代码、有 QA 测功能。你只需要下一条指令，剩下的全部自动完成。

### 2. 多实例并行：Claude Code + Codex + Gemini

除了 Claude Code，oh-my-claudecode 还能同时调度 Codex（OpenAI）和 Gemini（Google）的 Agent 实例，不同模型擅长不同领域：

  • Codex：擅长代码审查和安全分析
  • Gemini：适合 UI 设计和文档编写
  • Claude Code：擅长通用编程任务

比如输入一条指令：

/autopilot: build a REST API for managing tasks

它会自动规划出数据模型怎么设计、API 路由怎么写、谁来写测试用例，然后分给不同的 Agent 同时开工，最后汇总结果。

## 二、安装与核心指令 ### 1. 安装：一行命令搞定

oh-my-claudecode 的安装非常简单，不需要学新语法，直接在 Claude Code 里运行：

/plugin marketplace add https://github.com/Yeachan-Heo/oh-my-claudecode

安装完成后，重启 Claude Code 即可使用。

图 | Claude Code 运行 oh-my-claudecode 指令示意

### 2. 核心指令一览

安装完成后，以下是几个最实用的指令：

  • /autopilot：描述你的需求，自动拆解成子任务并执行
  • /deep-interview：用苏格拉底式提问帮你澄清需求，在动手前先把思路理清楚
  • /team N:executor "任务"：启动 N 个并行 Agent 执行特定任务
/team 3:claude "fix all TypeScript errors"

↑ 示例：启动 3 个 Claude Code 实例同时修复 TypeScript 错误

### 3. 适用场景

oh-my-claudecode 不是银弹，它最适合以下场景：

  • 大型重构项目：多模块并行改写，比一个人逐个改效率高得多
  • 安全代码审计：多个 Agent 分工检查不同模块，比单一审计更全面
  • PR Review：Codex 审查架构，Gemini 检查 UI，Claude Code 验证逻辑
  • 复杂需求澄清：用 /deep-interview 功能在动手前先理清思路
## 三、总结
oh-my-claudecode 解决的是一个很实在的问题：AI 编程工具很强，但一个人干活有上限。当你能指挥一个团队，各有所长、并行开工，效率就不再是线性增长，而是协同放大。

这个项目目前 Stars 突破 19k，说明社区对"多 Agent 协同"方向的认可。如果你也在用 Claude Code 或 Codex，不妨试试这个扩展，说不定会打开新世界。
