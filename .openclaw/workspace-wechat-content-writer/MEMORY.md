## 一、模板执行铁律（每次写稿前必读）

### tech-html-template-1.md 核心要点
- 标题：14-26字，专业清晰
- 导语：1-2段，每段≤3行
- 结构：导语 → 一、二级标题 → 三级标题 → 正文 → Plain Text代码块 → 图片 → 引用/列表 → 总结
- 代码块：**必须用 Plain Text 格式**，禁止 blockquote
- 图片：**必须上传微信CDN**，用 mmbiz.qpic.cn 永久链接，禁止外部图片链接
- 字数：900-1100字

### tech-html-acceptance-standards-1.md 阶段3核心要求
- 正文≥800字（实际执行以900-1100为准）
- 无事实错误、无逻辑漏洞
- 代码/命令可复现，无乱写
- 无错别字、无病句
- 每段≤120字、≤3行

## 二、高频格式错误记录（持续更新）
| 错误类型 | 错误描述 | 正确做法 |
|----------|----------|----------|
| Markdown输出 | 把Markdown当成HTML输出，公众号无法解析 | 必须输出HTML+内联样式 |
| 代码块格式错误 | 使用blockquote代替Plain Text代码块 | 使用 `<pre class="code-snippet">` + Plain Text |
| 图片链接失效 | 使用外部图片链接，推送后无法显示 | 必须上传微信CDN获取mmbiz.qpic.cn永久链接 |
| 配图堆叠文末 | 3张配图全部放在文末 | 均匀分布在开头/中间/结尾 |
| 字数超标 | 写了1250字，超出1100上限 | 严格控制在900-1100字 |
